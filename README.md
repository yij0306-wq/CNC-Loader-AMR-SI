# AMR 물류 최적화 시뮬레이터 (V42 Final 11)

본 저장소는 자율이동로봇(AMR)의 최적 투입 대수를 산정하고 설비별 물류 대기 지연(Bottleneck) 현상을 다각도로 평가하기 위한 디지털 트윈 기반 시뮬레이터입니다.
추후 유지보수 및 추가 개발 시 혼선이 없도록 시뮬레이션의 핵심 룰(Rule)과 종합 리포트의 데이터 산정 방식을 정리합니다.

---

## 1. ⚙️ 시뮬레이터 핵심 로직 및 룰 (Rules)

| 분류 | 규칙 명칭 | 상세 내용 및 기준 |
| :--- | :--- | :--- |
| **로더 생산** | 48개 배출 로직 | 각 CNC 로더 트레이 1칸당 6개의 제품이 적재되며, 트레이 8칸(총 48개)이 꽉 차면 즉시 배출(Eject)됨. |
| **로더 버퍼** | 3개 여유 버퍼 허용 | 로더가 48개를 배출한 직후에도 3개의 여유 버퍼(1.5트레이분 아님, 낱개 3개) 공간이 있어 장비는 멈추지 않고 가동됨. |
| **대기 지연** | 글로벌 대기 시간 기준 | 로더가 48개를 다 채우고 추가로 **여유 버퍼 3개까지 모두 꽉 찬 상태**에서만 장비가 물리적으로 멈추며, 이때부터 대기 지연(Wait Time) 타이머가 작동함. 개별 로더뿐 아니라 '시스템 글로벌 대기 시간'도 이 조건에서만 누적됨. |
| **AMR 호출** | 사전 호출 (Pre-Eject) | 로더의 생산이 완전히 끝나기 전에 미리 지정된 트레이 단수(예: 7단)에 도달하면 즉시 AMR에 콜(Call) 신호를 보내 로봇이 미리 대기할 수 있도록 함. |
| **AMR 제어** | 복귀 시간 (Return Time) | 각 AMR마다 설정된 최소 현장 대기시간(예: 1.5h)이 지나고 콜 신호가 없으면, 유휴 상태로 간주하고 강제로 충전소로 복귀함. |
| **AMR 속도** | 영역별 속도 이원화 | 수평 메인 통로는 0.8m/s, 입/출차용 수직 통로는 안전을 위해 0.4m/s의 감속 속도가 적용됨. |
| **AMR 회피** | 교차로 회피 (Siding) | 좁은 메인 통로나 로더 진입로에서 두 AMR이 마주치면 후행 진입 로봇이 빈 공간(Siding)이나 비어있는 CNC 영역으로 후진하여 양보 후 대기함 (물리적 제약 14초 지연 반영). |
| **AMR 도킹** | 정밀 도킹 딜레이 | AMR이 로더나 충전 베이에 진입/배출 시 방향 전환 및 리프트 승하강 등의 물리적 동작에 따른 30초의 딜레이가 적용됨. |

## 2. 📊 종합 리포트 지표 및 데이터 산정 방식

리포트는 매 프레임별 백그라운드 데이터 스니핑(Tracking) 방식을 사용하므로 시뮬레이터 본연의 룰이나 동선 로직에 부하를 주지 않습니다.

| 섹션 명칭 | 주요 지표 | 트래킹 방법 및 산출 수식 | AI 분석 기준 |
| :--- | :--- | :--- | :--- |
| **1. 생산성** | 시스템 평균 부하율 | 매 프레임의 `loadFactor`의 평균값. | `95%` 초과 시 과부하 셧다운 위험 경고 |
| **1. 생산성** | 설비 총 다운타임 | 전체 로더 중 가장 오래 멈춰있던 로더의 `cumulative_wait`이 아니라 전체 장비 멈춤을 글로벌 타임라인에서 합산 (이중 누적 배제). | 전체 시간 대비 비중이 높을 경우 이송 능력 한계로 규정 |
| **2. 활용률** | 전체 평균 활용률(%) | `(작업+이동 시간) / (작업+이동+대기+충전+회피시간) * 100` | 권장치: `80% ~ 90%`<br>`92%` 초과 시 충전 부족, `50%` 미만 시 과잉 투입 |
| **2. 활용률** | 상태별 유휴 분석 | AMR 내부 변수 `active_time`, `idle_time`, `charging_time`에 각 프레임 단위 초(Sim dt)를 누적. | - |
| **3. 배터리** | 최저 배터리율(SOC) | 충전 중이 아닐 때 소모되는 배터리 총량(초당 차감)을 실시간 추적하여 가장 낮게 떨어졌을 때의 %값(`min_soc`) 저장. | `20%` 미만 도달 시 붉은색 알림 및 충전 스케줄 조정 권고 |
| **3. 배터리** | 기회 충전 횟수 | 로봇 상태가 `CHARGING`으로 넘어가는 순간의 횟수(`charge_count`) 총합. | 잦은 복귀로 작업에 지장을 주면 대수 추가 권고 |
| **4. 트래픽** | 교착/회피 발생량 | 로봇 상태가 `EVADING_WAIT`에 진입한 총 횟수. | 회피 횟수가 과다하면 병목으로 진단 |
| **4. 트래픽** | 트래픽 체증 누적 시간 | `EVADING_WAIT`, `EVADING_TO_X` 등 회피 기동에 소모된 프레임 시간 누적. | 체증 시간 `1800초(30분)` 초과 시 Dual Lane(2라인) 모드 도입 권고 |
| **4. 트래픽** | 총 주행 거리 (km) | `moveTowards()` 동작 시 X/Y 픽셀 변화량을 실제 미터(1m = 38.99px)로 환산 후 누적. | 경로 비효율 측정용 데이터 |

## 3. 🚫 개발 및 유지보수 시 절대 금지 규칙
1. **레이아웃 유지:** `index.html` 내의 버튼 위치, 캔버스 비율, 조작 패널 크기 및 레이아웃을 임의로 변경하지 마십시오.
2. **시뮬레이션 로직 보존:** 로봇의 경로 탐색(`moveTowards`), 상태 전이 로직, 대기시간 타이머 등 코어 규칙은 물리 엔진과 직결되어 있으므로 절대 구조를 수정하거나 간소화하지 마십시오. 새로운 지표가 필요하다면 기존 방식과 동일하게 "백그라운드 누적 트래킹" 변수만 추가하십시오.
