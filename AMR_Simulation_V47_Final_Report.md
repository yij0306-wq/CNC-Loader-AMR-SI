# AMR 물류이송 시뮬레이션 최종 개발 규칙 및 릴리즈 노트 (V47)

본 문서는 AMR 물류이송 시뮬레이션의 최신 업데이트 내역과 최종 적용된 시뮬레이션 규칙(Rule Table)을 정리한 문서입니다.

## 🚀 최신 업데이트 내역 (V46 & V47)

### 1. `Min` 모델 추가 및 일괄 적용 기능
- **고속 `Min` 모델 추가**: 개당 75초의 고속 가공 시간을 가지는 `Min` 모델이 신규 추가되었습니다.
- **드롭다운 옵션 및 전용 뱃지**: 각 로더의 모델 선택 드롭다운에 `Min`이 추가되었으며, 상단 시스템 대시보드 및 **시뮬레이션 종료 종합 리포트**에도 `Min` 기종 누적 배출량이 정상 표기되도록 연동되었습니다.
- **[일괄 Min 적용] 기능키 신설**: 대시보드 상단 우측(초기화 버튼 옆)에 배치된 버튼을 클릭하면, ON 상태인 모든 로더가 한 번에 `Min` 모델(75초)로 설정됩니다.
- **입/출고 존 할당**: AMR이 충전 베이와 겹치지 않는 위치에 `Min` 전용 입/출고 존(IN/OUT)을 안전하게 할당받았습니다.

### 2. 정밀도 보장 **즉시 완료(Instant Complete)** 및 **고속 재생** 개선
- **즉시 완료 오차 차단**: 배속과 상관없이 즉시 완료 실행 시 내부 시뮬레이션 물리엔진을 **강제 1배속 연산(0.1초 고정 타임스텝)**으로 수만 번 반복 연산하여 현실과 100% 동일한 정밀도의 대기시간 및 생산량을 도출합니다.
- **고속 재생 일치화 (Time Slicing)**: 100배속~300배속 등 초고속 재생 시 한 프레임에 뭉텅이로 주어지는 시간(예: 4.8초)을 물리 엔진에 바로 넣지 않고, 즉시완료와 동일하게 **최대 0.1초 단위로 잘게 쪼개어 연산(Time Slicing)**합니다. 이로써 고속 재생 모드의 결과가 즉시완료 결과와 단 하나의 오차 없이 100% 일치하게 됩니다.
- **% 진행률 표시**: 브라우저 멈춤 현상(프리징)을 방지하기 위해 비동기 분할 연산(Chunking)을 도입하였으며, 연산 도중 버튼에 `⚡ 45.0% 완료...` 처럼 실시간 퍼센테이지를 표시합니다.

### 3. UI 및 안정성 픽스
- **폴더명 기반 타이틀 동기화**: `script.js`가 현재 브라우저 URL을 읽어들여 바탕화면 폴더의 이름(예: `Layout_V47_VinaLogic`)을 자동으로 탭 이름과 메인 좌측 상단 이름에 동기화합니다.
- **그룹명 세로쓰기 반전 수정**: `시뮬레이션 조작`, `알고리즘 제어`, `시스템 환경` 폴더 그룹명의 글자가 뒤집혀 보이던 CSS 버그를 똑바로 읽히도록 수정했습니다.

## 📋 최종 시뮬레이션 동작 규칙 (Rule Table)

1. **에이전트 기반 물리 엔진**: 하드코딩된 대기시간 수식이 아닌, AMR의 1) 주행 2) 충전 3) 정체 4) 교차로 양보 로직에 의해 자연적으로 대기시간이 발생합니다.
2. **우선순위 스케줄링**: 'DONE' 상태(생산 완료) 로더가 최우선 호출되며, 트레이 개수, 피스 개수 순으로 스케줄링됩니다.
3. **버퍼 및 대기시간 판정**: `DONE` 상태 이후 최대 3개의 잉여 생산물을 버퍼에 담으며, 이 버퍼마저 가득 차면 설비 정지로 간주하여 '빨간색 경고 표시'와 함께 실질적인 대기시간(LOADER WAIT)이 누적됩니다.
4. **배터리 운용**: 8시간 기준 배터리가 소비되며, 20% 이하에서 주황색 경고, 방전 시 강제 충전 모드로 전환됩니다.
5. **사전 배출**: 기능 설정에서 켜거나 끌 수 있으며, 8단 트레이 완성 전(예: 7단)에 미리 AMR을 호출해 대기시간을 최소화합니다.
