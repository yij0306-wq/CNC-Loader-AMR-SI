## 2. AMR 구동 및 물리 규칙 (Movement & Physics)

| 항목 | 상세 로직 및 적용 규칙 |
| :--- | :--- |
| **기본 이동 속도** | • 기본 주행 속도: **0.8 m/s**<br>• 시스템 전체 효율 80% 적용 ➡️ 실제 구동 속도: **0.64 m/s** |
| **세로 구간 감속** | • 수직 이동 구간(입/출차, 충전소 진입 등)에서는 안전을 위해 기본 속도의 50%인 **0.4 m/s** 로 감속 (실효율 적용 시 **0.32 m/s**) |
| **차량 회전 및 방향** | • X축, Y축 이동 벡터에 따라 차량의 방향(Rotation)이 시각적으로 즉시 동기화됨<br>• 세로 통행로(배출부) 진입 시에는 '게걸음' 모션(방향 유지 후 측면 이동) 규칙이 적용됨 |
| **교차로 감속 (1배속 강제)** | • 중앙 주 통행로(AMR Lane) 진입 시, 설정된 대수(예: 2대) 이상이 동시 진입하면 사고 방지를 위해 해당 차량들의 속도가 일시적으로 **1배속**으로 강제 제한됨 |

## 3. 배터리 및 충전 로직 (Battery & Charging)

| 항목 | 상세 로직 및 적용 규칙 |
| :--- | :--- |
| **배터리 총 용량** | • **8시간** (초 단위 변환 시 28,800초 지속) |
| **방전 및 충전 속도** | • 방전: 시뮬레이션 타임 기준 초당 1x 일정하게 소모<br>• 충전: 0%에서 100%까지 **2시간 완충** 기준 선형 충전 (방전 속도의 4배 빠름) |
| **충전소 복귀 규칙** | • 잔여 배터리 시간이 **1.5시간(약 18%)** 이하로 떨어지면, 현재 작업을 완료한 후 신규 콜을 받지 않고 즉시 충전소로 이동함 |
| **충전 중 대기** | • 충전소 도크에 진입한 후 **완충(100%)될 때까지** 빠져나오지 않고 대기함 |

## 4. 회피 및 충돌 방지 로직 (Collision Avoidance)

| 항목 | 상세 로직 및 적용 규칙 |
| :--- | :--- |
| **안전 감지 거리** | • 전방 위협 감지: UI에서 설정한 픽셀(기본 120px = **약 3m**) 거리 내에 장애물(타 AMR) 감지 시 정지 또는 회피 트리거 작동 |
| **회피 공간 (Siding)** | • 1열 및 2열 통행로의 각 로더(Loader) 사이사이 빈 공간에 가상의 회피존 동적 생성<br>• 13호기 우측 끝단에도 비상 회피용 여유 공간 추가 할당 |
| **회피 모드 (3가지)** | • **CNC 도킹존 회피:** 로더 앞 주차 공간만 사용하여 회피<br>• **회피존만 사용:** 로더 사이의 빈 Siding 공간만 사용하여 회피<br>• **둘 다 사용 (기본값):** 두 공간을 모두 활용해 가장 가까운 곳으로 능동적 회피 |
| **교착(Deadlock) 방지** | • 서로 마주 보는 상황 발생 시, 내부 우선순위에 밀린 AMR이 가장 가까운 빈 회피 공간(Y축 상/하 또는 X축 전/후)을 탐색하여 스스로 비켜줌 |

## 5. 물류 호출 및 작업 우선순위 (Task Dispatching)

| 항목 | 상세 로직 및 적용 규칙 |
| :--- | :--- |
| **콜(Call) 발생 조건** | • 로더의 생산 단수가 UI에서 지정한 '배출칸수(기본 8단)'에 도달하기 **1칸 전**에 선제적으로 AMR을 호출함 (사전배출 체크 시 즉시 호출) |
| **우선순위 룰 (3가지)** | • **적재차량 양보 (기본):** 짐을 실은(Loaded) AMR이 빈 차(Empty)보다 무조건 통행 우선권을 가짐<br>• **호기 번호순 우선:** 1호기가 2호기보다 무조건 우선 (서열 배차)<br>• **중앙통로 거리 우선:** 목표지점과 더 가까운 AMR에게 배차 및 통행 우선권 부여 |
| **투입/배출 도킹 위치** | • **투입 컨베이어 (하단):** AMR이 가로 상태(폭 20px)로 노란 박스 상단 외곽 라인에 1픽셀 오차 없이 딱 맞게 밀착 정차<br>• **배출 컨베이어 (우측):** AMR이 세로 상태(게걸음, 폭 20px)로 파란 박스 좌측 외곽 라인에 완벽히 밀착 정차 |

## 6. UI 및 데이터 시각화 (UI & Data Visualization)

| 항목 | 상세 로직 및 적용 규칙 |
| :--- | :--- |
| **시뮬레이션 배속** | • 1x 부터 최대 500x 까지 실시간 속도 조절 가능 |
| **대기시간 초과 경고** | • 로더의 누적 대기시간(지연) 발생 시, 로더 하단 텍스트 및 중앙 대시보드(LOADER WAIT)에 붉은색 폰트로 실시간 연동 표출 |
| **지연 상세 로그** | • 로더별 버튼 클릭 시 '시간별 배출 및 지연 히스토리' 모달창 제공 |
| **초기화 방식 (2가지)** | • **데이터 초기화:** 시뮬레이션 시계(Time)는 유지하면서 생산량, 로그, 상태만 초기화 (연속 시뮬레이션용)<br>• **시스템 초기화:** 시간, 배터리, AMR 위치까지 프로그램 구동 초기 상태(F5 효과)로 완벽히 리셋 |

## 7. 생산 택타임 및 물류 사이클 (Takt Time & Production Cycle)

| 항목 | 상세 로직 및 적용 규칙 |
| :--- | :--- |
| **장비별 사이클 타임 (C/T)** | • **M3 5X** (1~4호기): 제품 1개당 **150초**<br>• **M3 2ND** (5~8호기): 제품 1개당 **125초**<br>• **M3 UPPER** (9~16호기): 제품 1개당 **150초** |
| **적재 수량 및 트레이** | • **트레이 1단(칸)당 제품 6개** 씩 적재됨 |
| **배출 기준량 (Target)** | • 전역 배출 목표(기본값): **8단 (총 48개)**<br>• 1회 배출 시점 도달 시간: 150초 장비 기준 **정확히 2시간 (7,200초)**, 125초 장비 기준 **1시간 40분 (6,000초)** 소요 |
| **사전 배출 호출 로직** | • 로더별로 사전 배출 단수(기본 7단)를 개별 지정 가능<br>• UI에서 **[사전배출] 체크 시**, 배출 목표(8단)에 도달하기 전인 **7단 완료 시점에 미리 AMR을 호출(CALLING)** 하여 대기시간을 최소화함 |
| **상/하차 도킹 소요 시간** | • 로더 상차(Loading), 투입부 상차, 배출부 하차(Unloading) 시 모두 동일하게 **30초**의 시뮬레이션 타임을 소모한 후 이동을 재개함 |
